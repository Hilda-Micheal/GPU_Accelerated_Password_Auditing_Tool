# GPU_Accelerated_Password_Auditing_Tool

A CUDA-based MD5 password cracker benchmarked against a single-core Python CPU baseline. This is **Part 1** of a two-part series — Part 2 will extend the comparison to multi-core CPU execution.

---

## Overview

This project demonstrates the throughput advantage of GPU-accelerated password cracking over a single-threaded CPU implementation. It supports three attack modes and produces side-by-side benchmark plots for each.

The GPU kernel implements MD5 hashing entirely on-device (no per-candidate CPU↔GPU transfers), using CUDA constant memory for the target hash and atomic operations for early exit once a match is found.

---

## Attack Modes

| Mode | Description |
|---|---|
| **Brute-force** | On-GPU candidate generation across all lowercase strings up to a given length |
| **Dictionary** | Hashes every word in a wordlist (e.g. `rockyou.txt`) in parallel |
| **Rule-based** | Applies 8 mutation rules (capitalize, append `123`, leet-speak, etc.) to every dictionary word |

## Usage

### Run on Google Colab

1. Open the notebook in Colab and set the runtime to **GPU (T4)**.
2. Run all cells in order (Cells 1 → 12).
3. The notebook will:
   - Verify GPU availability
   - Write and compile `cracker.cu` and `md5.cuh`
   - Run all three attack modes against test hashes
   - Measure CPU single-core baseline throughput
   - Generate and save `benchmark.png`


## Benchmark Results (T4 GPU vs Single-Core Python)

| Attack | CPU (MH/s) | GPU (MH/s) | Speedup |
|---|---|---|---|
| Brute-Force (5-char, a–z) | ~1.03 | ~5,709 | ~5562× |
| Dictionary (14M words) | ~0.79 | ~2,618 | ~3316× |
| Rule-Based (14M × 8 rules) | ~0.77 | ~569 | ~741× |

> CPU baseline uses Python `hashlib` (single-core). GPU baseline uses a NVIDIA T4 (40 SMs). Your results will vary depending on Colab session hardware.

## Implementation Notes

- **MD5 kernel** (`md5.cuh`): A fully inlined `__device__` function that computes MD5 in registers — no shared memory, no global memory writes mid-computation.
- **Brute-force**: Each thread converts a unique integer index to a candidate string (base-N encoding) on-device — zero host-to-device data transfer per candidate.
- **Dictionary kernel**: Words are stored in a flat, coalesced GPU array (`words[i * MAX_WORD_LEN]`). Each thread hashes one word.
- **Rule kernel**: Launched with one block per word, `NUM_RULES` threads per block — each thread applies a different mutation.
- **Early exit**: A `__device__ int d_found` flag combined with `atomicExch` ensures all threads stop as soon as a match is found.
